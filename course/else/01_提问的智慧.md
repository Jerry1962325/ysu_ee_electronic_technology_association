## 提问的智慧

对于初学者来说，经常会在学习中提出问题。合适的提问方式有助于解答者快速定位问题的根源并给出恰当的答案

对于一个技术问题，提问前应该确保自己完成了以下步骤：

1. 根据提示或报错信息，在搜索引擎中搜索
2. 如果提示或报错信息是英文的，先确保你能看懂它，比如使用翻译软件进行翻译
3. 看看项目是否有附带文档或`README.md`文件，README文件是项目的自述文件
4. 如果你决定在互联网上公开提问，确保提问不会泄露你的个人隐私，比如公开用个人姓名命名的文件夹

现在请设想你是问题的解答者，你要做的是根据提问人的描述复现问题并找到解决方案

一个问题能稳定地重复发生，我们说这个问题是可复现的

如果一个问题是可复现的，那么这个问题已经解决了一半

因此，提问时的描述信息应该能方便解答人复现你的问题（无论是在脑海中还是在电脑上）

一般地来说，复现问题需要描述以下几点：

1. 程序的运行环境
2. 如果你参考了网上的文章或视频，附上链接
3. 代码或工程文件
4. 报错信息
5. 你期待的运行效果

程序的运行环境包括操作系统、编译器、编译选项、网络环境、集成开发环境

如果你不知道你的编译器和编译选项，就描述你的集成开发环境

针对网络问题，例如下载软件、驱动、代码等，描述你的网络环境，比如能否访问百度或谷歌

代码如果比较短的话，复制文本发送；如果代码或工程文件很大，你需要发送压缩包

反例是通过手机拍屏来提供代码

报错信息一般包括：错误定位、错误类型和错误细节

一般来说，提问所需的错误信息是错误类型和错误细节

当你需要截屏的时候，使用键盘上的`print screen(prt sc)`键来截图，或者使用QQ或微信自带的截图工具

！不要拍屏！

## 常用技术论坛

如果你想提问一个技术问题，以下是常用论坛：

1. 贴吧里对应技术的吧，例如C语言吧
2. 知乎
3. Github，如果代码在github上开源，你可以在对应的储存库中提issue（国外网站，英文）
4. Stack Overflow 是问写程序有关的问题。（国外网站，英文）
5. Server Fault 是问服务器和网管相关的问题。（国外网站，英文）
6. Super User 是问一些通用的电脑问题，如果你的问题跟代码或是写程序无关，只是一些网络连线之类的，请到这里。（国外网站，英文）
